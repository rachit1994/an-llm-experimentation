# Phase 2 test suite — attractor stability + encoder invariance (the headline)

Goal: (a) stored concepts denoise back to themselves (attractor stability), and (b) — the **headline** —
the *encoder* maps augmented inputs to the same pattern while keeping different concepts apart
(invariance **with** discrimination, so it cannot be won by collapse).

**Precondition:** Phase 1 G0 PASS. **Headline gate:** H4 (see
[`../METRICS-AND-GATES.md`](../METRICS-AND-GATES.md) §4).

---

## Cards

### T2.1 — Attractor stability (code-space denoising)
- Files: `qilm-train/src/metrics/stability.rs`, `qilm-train/tests/kat_stability.rs`
- Procedure: corrupt a stored code with `σ=0.10·meanstd(ξ)`; retrieve (T0.6); score hit if
  `sim(retrieved, clean) ≥ 0.90`.
- Done-when: `kat_stability` GREEN — hit-rate `≥ 0.90` over the fixture concept set, ≥5 seeds.

### T2.2 — Encoder-invariance metric with the discrimination margin
- Files: `qilm-train/src/metrics/invariance.rs`, `qilm-train/tests/nc_invariance.rs`
- Signature: `within(model, aug, rate)`, `between(model)`, `margin = within − between`.
- Done-when: `test_invariance_metric` GREEN on the planted-concept fixture — a **perfect** synthetic
  encoder (identity on planted id) reaches `within ≥ 0.90, between ≤ 0.30, margin ≥ 0.50`.
- Stays-red (the two controls that make this real):
  - `nc_collapsed_invariance` — `constant_encoder()` → `within≈between≈1`, `margin < 0.10`, H4 FAIL.
    *(Proves invariance can't be won by collapse — threat T4.)*
  - `nc_invariance_needs_Linv` — train with `no_invariance` → H4 FAILS (`margin < 0.50`).
    *(Proves the property is caused by `L_inv`, not incidental.)*
- Kills-mutant: define `margin = within` (drop discrimination) → `nc_collapsed_invariance` red.

### T2.3 — Known-answer end-to-end: planted-concept invariance
- Files: `qilm-train/tests/kae_invariance.rs`
- Procedure: train on `synth_concepts(...)`; evaluate on **held-out** concept ids.
- Done-when: `kae_invariance` GREEN — H4 gate PASS on held-out ids at the pre-registered aug rates
  {delete 10%, substitute 10%, case 50%, swap w=3}.
- Control inside the test: shuffling the planted ids must **destroy** the margin (`margin < 0.50`) —
  proving the encoder uses real planted structure, not an artifact of the metric.

### T2.4 — Run H4 on the real corpus, emit via report
- Done-when: `results/RESULTS.md` shows, from provenance-checked runs over ≥5 seeds,
  `within ≥ 0.90 ∧ between ≤ 0.30 ∧ margin ≥ 0.50` → **PASS**; else **NARROW to attractor-stability**
  (report the measured margin; do not slide the threshold).
