# Phase 5 test suite — unitary dynamics (copy / long-range)

Goal: decide whether a unitary layer buys long-range memory a free recurrence cannot, on the
known-solvable copy task. The result is **differential** (unitary beats `no_unitary` at long delay),
which is more robust than an absolute number.

**Precondition:** Phase 4 (complex). **Headline gate:** copy recall (see the phase doc
[`../PHASE-5-unitary-dynamics.md`](../PHASE-5-unitary-dynamics.md)).

---

## Cards

### T5.1 — Unitary layer stays unitary through training
- Files: `qilm-core/src/unitary.rs` (extend), `qilm-train/tests/prop_unitary_train.rs`
- Done-when: `prop_unitary_stays_unitary` GREEN — after N optimizer steps, `‖U†U − I‖ < 1e-4` still
  holds (Cayley) / by construction (Givens). *(Catches the bug where a "unitary" layer silently drifts
  non-unitary during training and the norm-preservation claim becomes false.)*

### T5.2 — Known-answer copy task, differential
- Files: `qilm-data/src/copy_task.rs`, `qilm-train/tests/kae_copy.rs`
- Done-when: `kae_copy` GREEN — unitary reaches `recall ≥ 0.99` at delay `T` where the `no_unitary`
  control (free linear recurrence, param-matched) stays `≤ chance + 0.05`. A *differential* pass:
  unitary must win **and** the control must fail, or the test is inconclusive (report as such, do not
  claim a win).

### T5.3 — Metal/Givens parity
- Done-when: `nc_determinism`-style check — Givens `U` on CPU and Metal agree within the documented
  Metal tolerance; the CPU number remains the source of truth (C3/C4).

## Phase-5 DoD checklist

- [ ] T5.1 unitarity preserved through training (proptest).

      `report`.
- [ ] T5.3 backend parity documented; headline number is the CPU number.
- [ ] If no differential gain: **drop the unitary layer** and record the measured recall curves — a
